id="quick-start"></a><br/>## 🚀 Quick Start<br/><br/>```powershell<br/># 1. Clone<br/>git clone https://github.com/Quadstronaut/EDWing.git<br/><br/># 2. Copy and edit the local config override<br/>Copy-Item example_configs\wing.conf.ps1.example wing.conf.ps1<br/># Edit wing.conf.ps1: set $cmdrNames, $sandboxieStart, $minEDLauncher, and any path overrides<br/><br/># 3. Launch<br/>.\Get-Wing.ps1<br/>```<br/><br/>> [!IMPORTANT]<br/>> `wing.conf.ps1` is gitignored — it holds your local paths and commander names. See [Configuration](#configuration) below.<br/><br/>---<br/><br/><a id="components"></a><br/>## 📦 Components<br/><br/>| File / Folder | What it does |<br/>|---|---|<br/>| `Get-Wing.ps1` | Main orchestrator — launches, positions windows, runs post-launch scripts |<br/>| `wing.conf.ps1` *(gitignored)* | Your local overrides for paths and feature toggles |<br/>| `example_configs/wing.conf.ps1.example` | Starter template for the above |<br/>| `autohonk/autohonk.py` | Watches Elite journal files; holds Primary Fire key after FSD jumps until `FSSDiscoveryScan` fires |<br/>| `clicker_scripts/cutscene.ps1` | Double-clicks each client window to dismiss the intro cutscene |<br/>| `clicker_scripts/continue-pg.ps1` | Clicks through Continue → Private Group → Launch on each client |<br/>| `clicker_scripts/MouseUtil.ps1` | Shared mouse helper (dot-sourced by the two scripts above) |<br/>| `input_broadcast.ps1` / `input_broadcast.py` | Experimental — relay keypresses to all Elite windows. **Not functional as of current master.** |<br/>| `installer_scripts/` | One-shot download-and-install scripts for MinEdLauncher, EDMC, EDEB, EDCoPilot |<br/>| `example_configs/` | Annotated config templates for MinEdLauncher |<br/><br/>---<br/><br/><a id="configuration"></a><br/>## ⚙️ Configuration<br/><br/>Copy `example_configs/wing.conf.ps1.example` to `wing.conf.ps1` in the repo root and edit it. Any variable set there overrides `Get-Wing.ps1` defaults.<br/><br/>**Minimum required changes:**<br/><br/>```powershell<br/># Your sandbox names — must match Sandboxie box names AND MinEdLauncher profile names<br/>$cmdrNames = @("CMDRFirst", "CMDRSecond", "CMDRThird", "CMDRFourth")<br/><br/># Paths on your machine<br/>$sandboxieStart = 'C:\Path\To\Sandboxie\Start.exe'<br/>$minEDLauncher  = 'C:\Path\To\Elite Dangerous\MinEdLauncher.exe'<br/>```<br/><br/>**Feature toggles:**<br/><br/>| Key | Default | Purpose |<br/>|---|---|---|<br/>| `launchEliteDangerous` | `$true` | Launch Elite clients |<br/>| `launchEDMC` | `$false` | Also launch EDMC per sandbox |<br/>| `launchEDEB` | `$false` | Launch EDEB for the first commander |<br/>| `skipIntro` | `$true` | Run `cutscene.ps1` after launch |<br/>| `pgEntry` | `$true` | Run `continue-pg.ps1` after cutscene skip |<br/>| `SeedCredentials` | `$true` | Back up and restore MinEdLauncher `.cred` files across sandbox wipes |<br/>| `StopCustomServicesAndProcesses` | `$false` | Kill background apps before launch |<br/><br/>### MinEdLauncher Configuration<br/><br/>See `example_configs/min-ed-launcher_min-ed-launcher.ini` and `example_configs/min-ed-launcher_settings.json` for annotated templates. MinEdLauncher expects one `[Profile.AccountN]` section per commander (email/password on first run, auto-login `.cred` file thereafter).<br/><br/>> [!WARNING]<br/>> Use `sandboxie-start` (the Sandboxie-Plus launcher binary) instead of the legacy `start` command referenced in older MinEdLauncher documentation.<br/><br/>---<br/><br/><a id="autohonk"></a><br/>## 🔊 AutoHonk<br/><br/>Monitors Elite journal files and automatically holds the Primary Fire key after an FSD jump, releasing when `FSSDiscoveryScan` completes (or after a configurable timeout). The Primary Fire key is read from your Elite key bindings file automatically; override with `--key` if needed.<br/><br/>Run one instance per sandbox:<br/><br/>```bash<br/># For sandboxed commanders<br/>python autohonk/autohonk.py --sandbox CMDRYourBoxName<br/><br/># For the primary (unsandboxed) commander<br/>python autohonk/autohonk.py<br/>```<br/><br/>**CLI flags:**<br/><br/>| Flag | Default | Description |<br/>|---|---|---|<br/>| `--sandbox` / `-s` | none | Sandboxie box name; resolves virtualised journal path |<br/>| `--window-filter` / `-w` | sandbox name | Substring to match in Elite window title |<br/>| `--delay` | `2.0` | Seconds after jump before firing |<br/>| `--max-duration` | `7.0` | Maximum seconds to hold the key |<br/>| `--key` | auto-detect | Override the key (e.g. `1`, `space`, `numpad_add`) |<br/>| `--verbose` / `-v` | off | Debug logging |<br/><br/>---<br/><br/><a id="window-positioning"></a><br/>## 🖥️ Window Positioning<br/><br/>Alt-monitor Elite clients are positioned at absolute pixel coordinates (negative X = left of primary monitor). The defaults in `Get-Wing.ps1` assume a specific multi-monitor layout.<br/><br/>> [!CAUTION]<br/>> Adjust `$eliteWindows` X/Y/Width/Height values in `wing.conf.ps1` for your setup. Clicker script coordinates (`cutscene.ps1`, `continue-pg.ps1`) are similarly pixel-absolute and **must be re-tuned** if your resolution or monitor arrangement differs.<br/><br/>---<br/><br/><a id="installer-scripts"></a><br/>## 🔧 Installer Scripts<br/><br/>`installer_scripts/` contains standalone PowerShell scripts to download and install each companion app. Edit the `$elitePath` / `$pathExtract` / `$DestinationPath` variables at the top of each script before running. These are one-shot helpers — they are **not** called by `Get-Wing.ps1`.<br/><br/>> [!WARNING]<br/>> `edeb.ps1` and `razzafrag_edcopilot-installer.ps1` install to hardcoded paths by default; edit before use. `razzafrag_edcopilot-installer.ps1` has a known limitation: silent MSI install does not work correctly.<br/><br/>---<br/><br/><a id="input-broadcast"></a><br/>## 🧪 Input Broadcast *(Experimental — Non-Functional)*<br/><br/>`input_broadcast.ps1` and `input_broadcast.py` attempt to relay keystrokes to all running Elite windows without requiring focus.<br/><br/>> [!CAUTION]<br/>> Both are present for reference but are **not working** on current master. See the scripts for inline notes on the approaches tried.<br/><br/>---<br/><br/>## 🔗 See Also<br/><br/>- [GitHub Wiki](https://github.com/Quadstronaut/EDWing/wiki) — full reference<br/>- [MinEdLauncher](https://github.com/rfvgyhn/min-ed-launcher)<br/>- [Sandboxie-Plus](https://sandboxie-plus.com/)<br/>
